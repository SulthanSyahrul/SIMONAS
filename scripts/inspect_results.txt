[Paragraph 541] Text: 'Arsitektur Sistem'

[Paragraph 542] Text: 'Arsitektur sistem yang diusulkan menggunakan model client-server dengan alur akses data yang terdiri dari Flutter Mobile App, REST/GraphQL API, Supabase Row Level Security (RLS), dan Supabase Data. Flutter Mobile App berfungsi sebagai antarmuka pengguna yang digunakan untuk mengakses fitur sistem, seperti login, jadwal, jurnal pembelajaran, absensi, nilai, tugas, serta administrasi pembelajaran.'

API berperan sebagai penghubung antara aplikasi mobile dengan layanan backend Supabase.'

[Paragraph 544] Text: 'Sebelum request dapat mengakses data, sistem terlebih dahulu melakukan validasi hak akses melalui Supabase Row Level Security (RLS). RLS digunakan untuk membatasi akses data berdasarkan role pengguna, seperti kepala sekolah, guru, siswa, dan kesiswaan. Dengan adanya validasi ini, setiap pengguna hanya dapat mengakses data yang sesuai dengan hak aksesnya.'

[Paragraph 545] Text: 'Apabila validasi hak akses berhasil, maka akses data diizinkan menuju Supabase Data. Supabase Data terdiri dari PostgreSQL Database untuk menyimpan data akademik dan Supabase Storage untuk menyimpan dokumen administrasi pembelajaran. Dengan arsitektur ini, sistem tidak hanya mendukung integrasi data akademik, tetapi juga menjaga keamanan akses data melalui mekanisme validasi berbasis role. Alur arsitektur sistem dapat dilihat pada Gambar 3. 12 Arsitektur Sistem.'

Kepala sekolah hanya diberikan hak SELECT (read-only) untuk memantau data tanpa risiko mengubah record secara tidak sengaja. [Paragraph 407] Text: '5.1 Metode Pengujian'

[Paragraph 408] Text: 'Pengujian yang disiapkan untuk aplikasi SinarBelajar adalah black-box testing. Metode ini berfokus pada pemeriksaan masukan dan keluaran fungsi tanpa melihat struktur internal kode. Pengujian black-box relevan untuk sistem informasi akademik karena setiap fitur dapat divalidasi berdasarkan skenario pengguna, seperti login, pemilihan role, input jurnal, input absensi, dan monitoring data. Referensi Muslimin dkk. (2020) serta Wulandari dkk. (2022) menunjukkan bahwa black-box testing dengan equivalence partitioning dapat digunakan untuk memeriksa validasi fungsi sistem informasi akademik.'

[Paragraph 409] Text: 'Selain black-box testing, pengujian tambahan yang disarankan adalah widget test untuk memastikan tampilan awal aplikasi muncul dengan benar, serta User Acceptance Testing (UAT) bersama pengguna sekolah. Pada naskah draft ini, tabel hasil pengujian disiapkan sebagai format pengisian. Status akhir perlu diisi setelah pengujian dilakukan pada perangkat dan data sekolah.'
